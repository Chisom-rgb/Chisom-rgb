HealthHive Project

TECH STACK:

MERN Stack
Frontend: React with JSX for UI
Axios for client–server communication
Backend: Node.js with Express for REST APIs
Database: MongoDB with Mongoose - flexible schema for user symptom submissions, AI outputs, and resource routing.
AI Integration:
Option 1: OpenAI API for text-based urgency classification
Option 2: Free open source AI models
Role-based access control: patient, provider, admin
OAuth 2.0 with JWT-based authentication
Password hashing and protected API routes

DEVOPS & DEPLOYMENT:

Docker & Docker Compose
GitHub Actions for CI/CD
Deployed on cloud hosting services

PROJECT DESCRIPTION:

HealthHive is a proof-of-concept wellness intake platform designed to explore whether AI can meaningfully assist users in understanding the potential urgency of their reported symptoms. Users will have personal dashboards to view previous submissions, track trends, and access suggested resources. AI analyzes submissions in real time, generating a clear urgency level with a short explanation. The platform will emphasize user safety, privacy, and ethical AI usage, with role-based access control to protect data.
