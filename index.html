03 — Projects

Things I've built

01 — Featured

HealthHive In Development

A proof-of-concept AI-powered wellness intake platform that classifies the urgency of self-reported symptoms — Emergency, High, Medium, or Low — using a language model. Designed as a case study exploring whether AI can produce consistent, explainable urgency classifications as a starting point for directing users to appropriate resources.

Users get personal dashboards to track submission history and trends. Role-based access control supports patient, provider, and admin roles. All AI outputs are treated as experimental results — the project is transparent about its POC scope, safety, and ethical boundaries.

React Node.js Express MongoDB OpenAI API JWT OAuth 2.0 Docker GitHub Actions RBAC

Tech Stack (MERN)

Frontend React + JSX, Axios
Backend Node.js, Express REST APIs
Database MongoDB + Mongoose
AI OpenAI API / open-source models

Security & Auth

Role-based access control (patient, provider, admin) with OAuth 2.0, JWT authentication, password hashing, and protected API routes.

DevOps

Docker & Docker Compose, GitHub Actions CI/CD pipeline, cloud deployment.

Research Hypothesis

"Can a language model, given self-reported symptoms, produce urgency classifications that are consistent, explainable, and useful as a starting point for resource routing?"
